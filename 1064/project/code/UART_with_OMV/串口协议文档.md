# 视觉模块串口通信协议文档

> 基于 `main.py` — 接收到 `0x01` 触发一次网格色块识别，回传地图数据。

## 1. 硬件层配置

| 参数 | 值 | 说明 |
|------|-----|------|
| 串口编号 | UART(12) | 对应硬件引脚 LPSR_06(TX) / LPSR_07(RX) |
| 波特率 | **19200** | |
| 数据位 | 8 | |
| 停止位 | 1 | |
| 校验位 | 无 | |

## 2. 通信模式：问答模式

```
主机（接收端）                    OpenMV（视觉端）
    |                                    |
    |-------- 0x01 (1字节) ------------>|  触发一次识别
    |                                    |  遍历所有网格进行色块检测
    |<------- 地图字符串 + \r\n --------|  回传识别结果
    |                                    |  蓝灯(LED3)闪一下
    |         (等待下一次触发...)         |
```

- 主机不发指令时，OpenMV 不做任何识别，不发送数据
- 每次收到 `0x01` 触发 **一次** 识别并回传

## 3. 指令与应答

### 3.1 触发指令

| 字段 | 值 | 说明 |
|------|-----|------|
| 内容 | `0x01` | 单字节 HEX |
| 长度 | 1 字节 | |

### 3.2 应答帧格式

```
{地图字符串}\r\n
```

| 部分 | 说明 |
|------|------|
| 地图字符串 | 192 个 ASCII 字符，按行优先排列 |
| 结束符 | `\r\n`（0x0D 0x0A） |

### 3.3 应答帧示例

```
-$.#*---------$.#*---...(共192字符)\r\n
```

## 4. 地图规格

| 参数 | 值 |
|------|-----|
| ROI 起始坐标 | (110, 60) |
| ROI 尺寸 | 167 × 127 像素 |
| 网格大小 | 10 × 10 像素 |
| 地图尺寸 | **16 列 × 12 行** |
| 地图总字符数 | **192** |
| 遍历顺序 | 行优先，每行从左到右 |

字符串索引 → 行列换算：

```
列坐标 col = index % 16
行坐标 row = index // 16    （整除）
```

即字符串前 16 字符为第 0 行，接下来 16 字符为第 1 行，以此类推共 12 行。

## 5. 地图字符定义

| 字符 | 含义 | LAB 阈值 (L_min, L_max, A_min, A_max, B_min, B_max) | 检测阈值 |
|------|------|---------------------------------------------------|---------|
| `$` | 箱子 | (65, 100, -60, -24, 9, 127) | pixels≥20, area≥20 |
| `.` | 目的地 | (0, 100, 27, 127, -68, 127) | pixels≥20, area≥20 |
| `#` | 墙 | (40, 100, -30, 30, -71, 4) | pixels≥**40**, area≥**40** |
| `*` | 英雄 | (0, 100, -128, -26, -128, 46) | pixels≥20, area≥20 |
